\DoxyCodeLine{36 \textcolor{comment}{   *    Cache Sub-\/System}}
\DoxyCodeLine{37 \textcolor{comment}{   *}}
\DoxyCodeLine{38 \textcolor{comment}{   * <Abstract>}}
\DoxyCodeLine{39 \textcolor{comment}{   *    How to cache face, size, and glyph data with FreeType\string~2.}}
\DoxyCodeLine{40 \textcolor{comment}{   *}}
\DoxyCodeLine{41 \textcolor{comment}{   * <Description>}}
\DoxyCodeLine{42 \textcolor{comment}{   *   This section describes the FreeType\string~2 cache sub-\/system, which is used}}
\DoxyCodeLine{43 \textcolor{comment}{   *   to limit the number of concurrently opened @FT\_Face and @FT\_Size}}
\DoxyCodeLine{44 \textcolor{comment}{   *   objects, as well as caching information like character maps and glyph}}
\DoxyCodeLine{45 \textcolor{comment}{   *   images while limiting their maximum memory usage.}}
\DoxyCodeLine{46 \textcolor{comment}{   *}}
\DoxyCodeLine{47 \textcolor{comment}{   *   Note that all types and functions begin with the `FTC\_' prefix.}}
\DoxyCodeLine{48 \textcolor{comment}{   *}}
\DoxyCodeLine{49 \textcolor{comment}{   *   The cache is highly portable and thus doesn't know anything about the}}
\DoxyCodeLine{50 \textcolor{comment}{   *   fonts installed on your system, or how to access them.  This implies}}
\DoxyCodeLine{51 \textcolor{comment}{   *   the following scheme:}}
\DoxyCodeLine{52 \textcolor{comment}{   *}}
\DoxyCodeLine{53 \textcolor{comment}{   *   First, available or installed font faces are uniquely identified by}}
\DoxyCodeLine{54 \textcolor{comment}{   *   @FTC\_FaceID values, provided to the cache by the client.  Note that}}
\DoxyCodeLine{55 \textcolor{comment}{   *   the cache only stores and compares these values, and doesn't try to}}
\DoxyCodeLine{56 \textcolor{comment}{   *   interpret them in any way.}}
\DoxyCodeLine{57 \textcolor{comment}{   *}}
\DoxyCodeLine{58 \textcolor{comment}{   *   Second, the cache calls, only when needed, a client-\/provided function}}
\DoxyCodeLine{59 \textcolor{comment}{   *   to convert an @FTC\_FaceID into a new @FT\_Face object.  The latter is}}
\DoxyCodeLine{60 \textcolor{comment}{   *   then completely managed by the cache, including its termination}}
\DoxyCodeLine{61 \textcolor{comment}{   *   through @FT\_Done\_Face.  To monitor termination of face objects, the}}
\DoxyCodeLine{62 \textcolor{comment}{   *   finalizer callback in the `generic' field of the @FT\_Face object can}}
\DoxyCodeLine{63 \textcolor{comment}{   *   be used, which might also be used to store the @FTC\_FaceID of the}}
\DoxyCodeLine{64 \textcolor{comment}{   *   face.}}
\DoxyCodeLine{65 \textcolor{comment}{   *}}
\DoxyCodeLine{66 \textcolor{comment}{   *   Clients are free to map face IDs to anything else.  The most simple}}
\DoxyCodeLine{67 \textcolor{comment}{   *   usage is to associate them to a (pathname,face\_index) pair that is}}
\DoxyCodeLine{68 \textcolor{comment}{   *   used to call @FT\_New\_Face.  However, more complex schemes are also}}
\DoxyCodeLine{69 \textcolor{comment}{   *   possible.}}
\DoxyCodeLine{70 \textcolor{comment}{   *}}
\DoxyCodeLine{71 \textcolor{comment}{   *   Note that for the cache to work correctly, the face ID values must be}}
\DoxyCodeLine{72 \textcolor{comment}{   *   *persistent*, which means that the contents they point to should not}}
\DoxyCodeLine{73 \textcolor{comment}{   *   change at runtime, or that their value should not become invalid.}}
\DoxyCodeLine{74 \textcolor{comment}{   *}}
\DoxyCodeLine{75 \textcolor{comment}{   *   If this is unavoidable (e.g., when a font is uninstalled at runtime),}}
\DoxyCodeLine{76 \textcolor{comment}{   *   you should call @FTC\_Manager\_RemoveFaceID as soon as possible, to let}}
\DoxyCodeLine{77 \textcolor{comment}{   *   the cache get rid of any references to the old @FTC\_FaceID it may}}
\DoxyCodeLine{78 \textcolor{comment}{   *   keep internally.  Failure to do so will lead to incorrect behaviour}}
\DoxyCodeLine{79 \textcolor{comment}{   *   or even crashes.}}
\DoxyCodeLine{80 \textcolor{comment}{   *}}
\DoxyCodeLine{81 \textcolor{comment}{   *   To use the cache, start with calling @FTC\_Manager\_New to create a new}}
\DoxyCodeLine{82 \textcolor{comment}{   *   @FTC\_Manager object, which models a single cache instance.  You can}}
\DoxyCodeLine{83 \textcolor{comment}{   *   then look up @FT\_Face and @FT\_Size objects with}}
\DoxyCodeLine{84 \textcolor{comment}{   *   @FTC\_Manager\_LookupFace and @FTC\_Manager\_LookupSize, respectively.}}
\DoxyCodeLine{85 \textcolor{comment}{   *}}
\DoxyCodeLine{86 \textcolor{comment}{   *   If you want to use the charmap caching, call @FTC\_CMapCache\_New, then}}
\DoxyCodeLine{87 \textcolor{comment}{   *   later use @FTC\_CMapCache\_Lookup to perform the equivalent of}}
\DoxyCodeLine{88 \textcolor{comment}{   *   @FT\_Get\_Char\_Index, only much faster.}}
\DoxyCodeLine{89 \textcolor{comment}{   *}}
\DoxyCodeLine{90 \textcolor{comment}{   *   If you want to use the @FT\_Glyph caching, call @FTC\_ImageCache, then}}
\DoxyCodeLine{91 \textcolor{comment}{   *   later use @FTC\_ImageCache\_Lookup to retrieve the corresponding}}
\DoxyCodeLine{92 \textcolor{comment}{   *   @FT\_Glyph objects from the cache.}}
\DoxyCodeLine{93 \textcolor{comment}{   *}}
\DoxyCodeLine{94 \textcolor{comment}{   *   If you need lots of small bitmaps, it is much more memory efficient}}
\DoxyCodeLine{95 \textcolor{comment}{   *   to call @FTC\_SBitCache\_New followed by @FTC\_SBitCache\_Lookup.  This}}
\DoxyCodeLine{96 \textcolor{comment}{   *   returns @FTC\_SBitRec structures, which are used to store small}}
\DoxyCodeLine{97 \textcolor{comment}{   *   bitmaps directly.  (A small bitmap is one whose metrics and}}
\DoxyCodeLine{98 \textcolor{comment}{   *   dimensions all fit into 8-\/bit integers).}}
\DoxyCodeLine{99 \textcolor{comment}{   *}}
\DoxyCodeLine{100 \textcolor{comment}{   *   We hope to also provide a kerning cache in the near future.}}
\DoxyCodeLine{101 \textcolor{comment}{   *}}
